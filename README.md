# Happy Hardware (Team 27)

> Happy Hardware is an e-commerce platform for selling computer components. It's designed for students, professionals, and gamers who want high performance on a budget but feel intimidated by the complexity of custom PC building.We differentiate ourselves with a welcoming UI and comprehensive support, including a parts picker and detailed guides to help users build their first PC with confidence.

---

##  Key Features

* **Storefront:** A welcoming homepage that displays current best-sellers.
* **Dynamic Navigation:** A fixed navigation bar that expands on hover for easy access to all product categories (CPU, GPU, RAM, etc.).
* **Part Picker:** A core feature that allows users to select and compare hardware components, automatically validating compatibility between parts.
* **Shopping Basket:** A full checkout process with secure card fields and payment authentication.
* **Product Pages:** Detailed pages with product specifications, descriptions, and an "Add to Basket" button.
* **User Accounts:** Users can create an account, log in, and view their profile with past order history.
* **AI Chatbot:** A toggleable chatbot to answer user questions, using pre-defined answers or generating specific responses. 

---

##  Tech Stack

* **Backend:** [**Laravel**](https://laravel.com/) - A robust PHP framework used for a secure structure and rapid development.
* **Frontend:** [**Vue.js**](https://vuejs.org/) - A component-based JavaScript framework for creating a dynamic and maintainable user interface.
* **Styling:** [**Tailwind CSS**](https://tailwindcss.com/) - A utility-first CSS framework for quickly prototyping and building responsive designs.

##  Design & Planning

The project was planned extensively before development, including a site map, database schema, and full UI mockups.

### Site Map
The site map outlines the page hierarchy and user flow, starting from the central "HOME" page and branching into main categories like "Store," "Account," and "Other Pages."

### Database Schema
Our database schema defines all entities (like `products`, `customer`, `orders`), their attributes, and the relationships between them.

### Wireframes & Mockups
We created low-fidelity wireframes and high-fidelity Figma mockups to visualize the website's layout and user experience before writing code.

##  Project Management & Contributing

This project is managed using the **Kanban (Agile)** methodology, chosen for its flexibility in handling changing requirements.

* **Task Management:** We use a [**Trello board**](https://trello.com/invite/b/68fd158dd905f7ff02d91278/ATTIab587ba60de33a892a285027631665ab7F2D0069/happy-hardware) to visualize our workflow in "To Do," "In Progress," "Review," and "Done" columns.
* **Code Collaboration:** All code is managed on here on GitHub.
* **Workflow:** For every Trello task, a corresponding GitHub branch is created. All code must pass a **Pull Request (PR) review** by another team member before being merged to ensure code quality.
